### 3. ProjectDetailPage（STAR结构项目详情）
**文件**: `project-detail.tsx`

**STAR结构**:
```
01. Situation (背景与挑战)
    - 项目背景
    - 核心挑战
    - 为什么重要

02. Task (我的角色)
    - 负责什么
    - 团队规模
    - 协作方式

03. Action (方法与过程)
    - Step 1: 用户洞察
    - Step 2: 方案设计
    - Step 3: 执行优化
    - Step 4: 系统搭建

04. Result (核心成果)
    - 量化指标（6个数据卡片）
    - 定性反馈
    - 成果截图

05. Insights (关键洞察)
    - 我学到了什么
    - 如果重来会怎么做
```

**数据配置**:
```typescript
const projectData: ProjectDetail = {

  task: {
    title: "我的角色与职责",
    content: "...",
    myRole: "活动策划与执行负责人",
  },
  
  // ... 完整结构见代码

```

**使用方式**:

## 💬 常见问题

### Q: 需要做3个完全不同的网站版本吗？
A: 不需要。使用职位切换器，一个网站支持多个版本。

### Q: 攀岩路线和STAR详情页哪个更重要？
A: 都重要！攀岩路线是独特性（吸引眼球），STAR详情页是专业性（建立信任）。

### Q: 简历PDF和网站内容要一致吗？
A: 是的。网站是简历的延伸，但可以有更多细节。

### Q: 博客是必须的吗？
A: 不是必须，但强烈推荐。3-5篇精选文章能大幅提升专业度。
